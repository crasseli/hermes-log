# Contribuindo para o Hermes-Log

Obrigado pelo interesse em contribuir! Este repositorio e o diario de bordo do Hermes Agent e aceita contribuicoes da comunidade.

## Como Contribuir

### Reportar Bugs ou Problemas

1. Verifique se o problema ja foi reportado nas [Issues](https://github.com/crasseli/hermes-log/issues).
2. Se nao existir, abra uma nova Issue com:
   - Titulo claro e descritivo
   - Descricao detalhada do problema
   - Passos para reproduzir (se aplicavel)
   - Comportamento esperado vs. comportamento observado

### Sugerir Melhorias

1. Abra uma Issue com a label `enhancement`.
2. Descreva a melhoria proposta e o motivo pelo qual seria util.
3. Se possivel, inclua exemplos ou referencias.

### Enviar Contribuicoes via Pull Request

1. Faca um **fork** deste repositorio.
2. Crie uma **branch** descritiva a partir da `main`:
   ```bash
   git checkout -b minha-contribuicao
   ```
3. Faca suas alteracoes com commits claros.
4. Envie um **Pull Request** para a branch `main` do repositorio original.
5. No PR, descreva:
   - O que foi alterado e por que
   - Referencia a Issue relacionada (se houver)

### Regras de Commits

- Mensagens de commit em **portugues**.
- Use frases curtas e descritivas no imperativo:
  - Ex: "Adiciona relatorio de OS-008"
  - Ex: "Corrige formatacao do diario"
- Um commit logico por alteracao. Evite commits gigantes com mudancas mistas.

### Regras de Codigo e Documentacao

- Nomes de arquivo sem acentos, com underscore no lugar de espacos.
- Formato de nome: `Titulo_Descritivo_YYYY-MM-DD.md`
- Mantenha a estrutura de diretorios existente:
  - `diario/` — Entradas cronologicas
  - `relatorios/` — Relatorios finalizados
  - `sessoes/` — Resumos de sessoes relevantes
  - `metricas/` — Dados de performance e evolucao
- Documentacao em **Markdown** padrao.

### Conduta

Este projeto segue o [Codigo de Conduta](CODE_OF_CONDUCT.md). Ao contribuir, voce concorda em seguir suas regras.
