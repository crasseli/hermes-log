# Relatório OS-003 — Diagnóstico Protocolo Recall

## Causa Raiz (3 camadas interdependentes)

**Camada 1 — Tool Gateway:** search_vault.py e vault_check.py não estão registrados como tools no gateway oficial. O orquestrador não consegue executá-los como primeira ação — eles só existem como scripts via terminal. A instrução do SOUL.md é fisicamente ininterpretável pelo orquestrador.

**Camada 2 — Infraestrutura quebrada:** sqlite-vec não está instalado no sistema. A busca vetorial semântica nunca funcionou. O sistema opera em modo degradado silencioso usando apenas FTS (Full Text Search) sem avisar.

**Camada 3 — Estado inconsistente:** 289 registros reais vs 153 reportados anteriormente. 3 scripts com erros de sintaxe (link_notes.py, populate_vault_from_sessions.py, show_backlinks.py). O sistema reporta saúde que não corresponde ao estado real.

## Evidências

| Achado | Evidência |
|--------|-----------|
| search_vault.py não é tool | Ausente do tool gateway, só em logs e código |
| sqlite-vec ausente | sqlite_vec.so não encontrado no sistema |
| Busca vetorial = 0 | vec_obsidian inacessível via SQL |
| Contagem inconsistente | 153 reportado vs 289 real |
| Scripts quebrados | 3 SyntaxError/IndentationError confirmados |

## Por que o protocolo é violado

O SOUL.md instrui: primeira ação = search_vault.py. O orquestrador recebe essa instrução mas search_vault.py não existe no tool gateway. Sem tool válida para executar, o orquestrador cai para o próximo recurso disponível: session_search. A violação não é desobediência — é impossibilidade técnica.
