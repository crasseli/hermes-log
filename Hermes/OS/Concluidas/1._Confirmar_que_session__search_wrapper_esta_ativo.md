## 1. Sumario Executivo

Esta Ordem de Servico define a implementacao do Pipeline Hierarquico de Recall no agente Hermes. O objetivo e garantir que qualquer busca por informacao siga a ordem: Vault Obsidian (FTS5) -&gt; SQLite de sessoes -&gt; Web Search, sendo a internet usada apenas como ultimo recurso quando os sistemas locais nao retornarem resultados relevantes.

Pergunta do usuario

| se 0 resultados relevantes

2. session\_search                &lt;- SQLite state.db

| se 0 resultados relevantes

v

3. web\_search\_tool()             &lt;- Internet (ULTIMO RECURSO)

**Funcao auxiliar a adicionar no topo do arquivo:**

def run\_vault\_check(query: str, threshold: int = 2) -&gt; dict:

try:

script = Path.home() / '.hermes/skills/obsidian/scripts/vault\_check.py'

if not script.exists():

return {'has\_results': False, 'count': 0, 'results': ''}

result = subprocess.run(

['python3', str(script), query, '--threshold', str(threshold)],

capture\_output=True, text=True, timeout=12

)

if result.returncode == 0:

return json.loads(result.stdout)

return {'has\_results': False, 'count': 0, 'results': ''}

except Exception:

return {'has\_results': False, 'count': 0, 'results': ''}
